Section 01

Product Vision

VanderBot is an AI-native creative workspace for the Vanderbilt Owen MBA program. It combines conversational AI with structured content creation — letting students and faculty generate documents, pitch decks, research reports, infographics, and web pages from a single mobile interface.

Not a chatbot. Not a note-taking app. A creative operating system powered by Claude.

Mode Switcher

Horizontal pill bar below top bar. Modes share the same chat thread but change the AI's behavior:

💬 Chat 🔬 Deep Research 📓 NotebookLM 📊 Infographic
ModeChip ColorBehavior
💬 ChatStandard Claude conversation
🔬 Deep ResearchMulti-source research with citations, longer responses
📓 NotebookLMCreate study guides, flashcards, audio summaries
📊 InfographicGenerate data visualizations from conversation context

Switching modes mid-conversation: the mode chip animates, the system prompt changes, but the conversation history is preserved.

Message Bubbles

  • User: right-aligned, blue gradient background, white text
  • Assistant: left-aligned, glass card background, dark text
  • Streaming: cursor blink animation, characters build progressively
  • Markdown: headers, bold, italic, code blocks, lists
  • Artifact cards: inline glass cards for generated content

Input Bar

  • Glass card at bottom with safe area padding
  • Text input: expandable, max 5 lines before scroll
  • Left: attachment (📎) + camera (📷) buttons
  • Right: send (↑) — blue when text present, dim when empty

5.3 — Chat Modes (Detailed)

Deep Research Mode 🔬

When activated, the AI searches multiple sources (web, academic, news) and returns structured reports with sections and citations. Shows a "Researching..." progress indicator with source count.

  • Source pills below response (clickable → opens URL)
  • Citation numbers [1] [2] [3] inline, tappable
  • Depth selector: Quick (3 sources) / Standard (8) / Deep (15+)

NotebookLM Mode 📓

Creates structured study materials: Study Guides, Flashcard Sets, Audio Overviews, FAQ, Timeline.

  • Output type selector: Study Guide / Flashcards / Audio / FAQ / Timeline
  • Flashcard view: swipeable card stack with flip animation
  • Audio player: inline waveform for generated audio summaries
  • "Add Source" button: paste URLs, upload PDFs, or reference prior chats

Infographic Mode 📊

Analyzes conversation data and generates visual infographics. Auto-detects chart types.

  • Style selector: Corporate / Minimal / Bold / Playful
  • Export: PNG, PDF, or embed in a Deck
  • Edit: tap to modify data points, colors, labels

Section 07

Interaction Patterns

7.1 Streaming Responses
  • Characters appear one at a time with a blinking cursor
  • Markdown rendered progressively (headers snap in, lists build)
  • Artifact cards slide in with spring animation when complete
  • "Stop generating" button appears during streaming
7.2 Haptic Feedback
  • Light tap: button presses, tab switches
  • Medium: mode switch, send message
  • Heavy: delete, publish
7.3 Gesture Support
  • Swipe right on chat bubble → reply/quote
  • Swipe left on file card → delete
  • Long press on message → copy, share, regenerate
  • Pull to refresh on file list
  • Pinch to zoom on page preview / deck slides
7.4 Loading States
  • Skeleton screens (not spinners) for content loading
  • Shimmer animation on placeholder cards
  • "Thinking..." indicator with 3 animated dots
7.5 Empty States
  • Home: "Start creating. Tap any card above."
  • Files: "Nothing here yet. Create something amazing."
  • Chat: Welcome message with suggested prompts
7.6 Error Handling
  • Toast notifications for transient errors (network, API)
  • Inline error cards for persistent issues
  • Retry buttons on failed generations
  • Offline mode: draft editing works, syncs when connected

Technical Architecture

API Layer
API Architecture 
Claude API (Anthropic)
├── Streaming SSE for chat
├── Vision (image upload → base64)
├── System prompts per mode:
│   ├── Chat: general assistant
│   ├── Research: structured report + citations
│   ├── NotebookLM: study materials
│   └── Infographic: data extraction + visualization

GitHub REST API
├── Push HTML pages to gh-pages branch
├── Commit history as version control

Google Drive API (optional)
├── Export docs to Google Docs
├── Export decks to Google Slides
└── Backup all files to Drive folder
Local Storage (Room Database)
Database Schema 
conversations
├── id (UUID)
├── title
├── mode (chat/research/notebook/infographic)
├── created_at, updated_at

messages
├── id (UUID)
├── conversation_id (FK)
├── role (user/assistant)
├── content (markdown)
├── artifacts (JSON — embedded docs/decks/pages)
├── created_at

documents
├── id (UUID)
├── type (doc/deck/page/infographic/research)
├── title
├── content (markdown/html/json)
├── status (draft/published/exported)
├── github_path (nullable)
└── created_at, updated_at

Section 11

Accessibility

  • Dynamic type support (system font scaling)
  • Minimum contrast ratio: 4.5:1 (WCAG AA)
  • Screen reader labels on all interactive elements
  • Reduce Motion: disables spring animations, parallax
  • Focus indicators for keyboard/switch navigation
  • Alt text on all generated images
